# Components: Card

Official docs: `https://picocss.com/docs/card`

Pico uses semantic `<article>` as its card primitive.

## Key points

- Plain `<article>` already renders as a card container.
- `<header>` and `<footer>` can be added inside the article for sectioning.
- This is one of Pico's clearest examples of semantic markup replacing component-specific classes.

## Syntax

```html
<article>
  <header>Header</header>
  Body
  <footer>Footer</footer>
</article>
```
